ASCII 编码

0 - 31及 127 是控制字符或通信专用字符。

32 - 126 是字符，其中 48 - 57 为 0 到 9 的十个阿拉伯数字，65 - 90 为 26 个大写英文字母，97 - 122 为 26 个小写英文字母，其余为标点符号和运算符号等。

<vector>

vector 的存储是自动管理的，按需扩张收缩。 vector 通常占用多于静态数组的空间，因为要分配更多内存以管理将来的增长。 vector 所用的方式不在每次插入元素时，而只在额外内存耗尽时重分配。分配的内存总量可用 capacity() 函数查询。额外内存可通过对 shrink_to_fit() 的调用返回给系统。 (C++11 起)

重分配通常是性能上有开销的操作。若元素数量已知，则 reserve() （void reserve( size_type new_cap );）函数可用于消除重分配。

vector 上的常见操作复杂度（效率）如下：

  • 随机访问——常数 O(1)
  • 在末尾插入或移除元素——均摊常数 O(1)
  • 插入或移除元素——与到 vector 结尾的距离成线性 O(n)

<set>

std::set 是关联容器，含有 Key 类型对象的已排序集。用比较函数 比较 (Compare) 进行排序。搜索、移除和插入拥有对数复杂度。 set 通常以红黑树实现。

在每个标准库使用比较 (Compare) 概念的场所，用等价关系确定唯一性。不精确地说，若二个对象 a 与 b 相互间既不比较大于亦不比较小于： !comp(a, b) && !comp(b, a) ，则认为它们等价。

std::set 满足容器 (Container) 、具分配器容器 (AllocatorAwareContainer) 、关联容器 (AssociativeContainer) 和可逆容器 (ReversibleContainer) 的要求。

小贴士

  1. 输入超大，想到 string，当做数使用想到s[i] - ‘0’，输出分类想到用集合映射

  2. 模式匹配用 map 输入，根据题目设置条件判断

  3. 求数据中的极值不用存，时刻想到什么时候该存数据(用最少的位数存最大信息量，存极小部分数据，剩下的都可以由存下的数据算出)，什么时候不用存

  4. 对很多结果进行保存索引稀疏数组num[v[i]] = result，v[i]是遍历的元素

  5. 预处理数组：当查询需遍历且是不止一次的操作时，是不是能方便地求得一个辅助数组，包含要查询的结果。
